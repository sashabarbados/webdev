git init - создание нового репозитория (в папке, которой работаем - скрытая папка .git)
Если нужно сбросить настройки git`a - удаляем эту папку внутри проекта, проект отвяжется от git репозитория и будет таким, какой есть в текущий момент

git status - смотрим, из чего состоит наш проект (показывает, что сейчас мы находимся на ветке master (ветка по умолчанию); ветка - это различные пути развития проекта)


git add * - добавить все файлы папки в проект (либо прописать каждое имя по отдельности)

git commit -m "commit 1" 
commit - слепок текущего состояния проекта (принято делать всегда с комментариями)
-m (message) - указываем комментарий к коммиту

далее на github создаем репозиторий и 

git remote add origin https://github.com/sashabarbados/имя_репозитория.git - добавить удаленный репозиторий, с которым git локальный будет связан

git push -u origin master - заливаем на сервер (удаленный репозиторий), ветку master текущие изменения (может потребоваться небольшая авторизация)

для того чтобы скачать проект с удаленного сервера на github:

git init
git remote add origin https://github.com/sashabarbados/имя_репозитория.git

git pull origin master - скачать данные с сервера


Если делаешь какие-то изменения в файле, чтобы залить их на сервер всегда юзай:
git status (для проверки)
git add <file> - тот, в котором внес изменения
